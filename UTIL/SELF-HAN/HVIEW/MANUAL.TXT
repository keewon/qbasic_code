                 I. 저작권 및 사용자의 의무

1. 'V3+' 프로그램과 관련 설명서들에 대한 모든 저작권과 지적
    소유권은 제작사인 '안철수컴퓨터바이러스연구소'가 소유하
    며, 저작권법과 컴퓨터 프로그램 보호법에 의해 보호됩니다.

2) 'V3+'를 정식 구입하지 않고  공동 업무(관공서, 기업 등 단
   체 사용)에 이용하거나 제작사의 허락 없이 상업적인 목적에
   절대로 사용할 수 없습니다.  또한 하드웨어나 소프트웨어를 
   판매할 때 판촉이나 직접적인 영리의 목적으로  복사해줄 수 
   없으며,  월간지나 단행본 등에 포함되는 디스켓에 복사해서
   배포할 수도 없습니다. 이러한 목적으로 사용하고자 할 경우
   에는 반드시 안철수컴퓨터바이러스연구소의 허락을 얻어야만
   합니다.

3. 'V3+' 프로그램과 관련되는 설명서 파일들의  파일 이름이나
   내용을 제작사의 허락없이 변형시킬 수 없습니다.

4. 통신을 통해서 배포할 경우에는 제작사가 제공한 프로그램과
   모든 관련 설명서 파일들을 같이 배포해야만 합니다. 즉, 제
   작사가 제공한 파일들 중에서 일부를 누락시킬 수 없으며 제
   작사가 제공하지 않은 파일들과 함께 압축 배포할 수 없습니
   다. 또한, 다른 프로그램과 함께 압축되어 배포하는 것도 허
   락되지 않습니다.


                 II. 제작사의 연락처

속하게 보내 주시기 바랍니다. 신종 컴퓨터 바이러스 여부를 확
인한 후  신속한 백신 프로그램 버전업으로 사용자 피해를 최소
화할 것입니다.

                 III. 사용하기 전에

V3+에 포함되어 있는 V3.EXE는  지금까지 국내에서 발견된 컴퓨
터 바이러스들을 진단, 치료하는 프로그램이고, V3RES.EXE는 기
억 장소에 상주하면서 해당 컴퓨터 바이러스들을 예방하는 프로
그램입니다. 또한 V3BACKUP.EXE는 부트 바이러스에 감염된 부트
영역의 샘플을 간단하게 채취할 수 있는 유틸리티입니다.

백신 프로그램의 버전(Version)은 국내에서 발견된 예방, 진단,
치료 가능한 컴퓨터 바이러스의 숫자를 나타냅니다.  예를 들어
버전 1390은 1390종의 컴퓨터 바이러스를 진단, 치료, 예방할 수
있다는 것을 뜻합니다.

그런데  V3+는 지금까지 국내에서 발견된 컴퓨터 바이러스에 대
해 예방, 진단, 치료할 수 있지만 새로운 컴퓨터 바이러스는 진
단할 수 없습니다. 즉, 새로운 컴퓨터 바이러스는 진단할 수 없
습니다.  

따라서, 컴퓨터 바이러스에 대해서 효과적으로 대처하기 위해서
는 백신 프로그램만 믿을 것이 아니라 중요한 자료를 복사해 두
는 백업(backup) 등의 다른 수단을 병행해 사용하는 것이  바람
직합니다.

또한, V3+로 컴퓨터 바이러스에 감염된 프로그램을 치료할 경우
대부분 원래대로 복구가 가능하지만  신종 변형 바이러스, 겹쳐
쓰기 바이러스, 중복 감염 등의 경우에는 복구가 되지 않으므로
치료하기 전에 데이터와 감염된 파일을 반드시 백업해두고 치료
하시기 바랍니다.


* V3+의 사용 조건

V3.EXE, V3RES.EXE는 도스 3.2 이상이 설치되어 있는 IBM-PC 호
환 컴퓨터에서 사용이 가능하고,  MS-DOS 3.2 이상의 버전과 호
환되는  DR-DOS, PC-DOS, Novell-DOS, 4-DOS, N-DOS에서 사용할
수 있습니다.

V3RES.EXE는 일반 기억 장소(conventional memory)에  상주시킬
수 있으며, 286 이상 컴퓨터에서는 상위 기억 장소(upper memo-
ry)를 만든 경우에는 상위 기억 장소에 상주시킬 수 있습니다.   

단, 상위 기억 장소에 상주시키려면  연속된 약 30KB의 상위 기
억 장소가 필요합니다. 그러나 기억 장소에 상주되면 기억 장소
의 종류에 관계없이 약 23KB의 기억 장소를 차지합니다.


                 IV. V3+의 설치 방법

V3+를 설치하려면, 제일 먼저 원본 플로피 디스크를 드라이브에
넣고 등록판의 경우는 설치 프로그램인 INSTALL.EXE를 실행하면
설치할 수 있습니다.

그러나 비등록판은 설치 프로그램이 없으므로 설치를 원하는 디
스크에 디렉토리를 만든 후 관련 파일들을 복사하여 사용하시기 
바랍니다. 예를 들어, A 드라이브에  V3+ 프로그램 디스크가 존
재할 때 C 드라이브의 \V3 디렉토리에 V3+를 설치하고자 한다면
다음과 같은 방법을 사용하면 됩니다.

  A:\>C:                <- C 드라이브로 옮겨간다.
  C:\>MD V3             <- \V3 디렉토리를 만든다.
  C:\>CD V3             <- \V3 디렉토리로 옮겨간다.
  C:\V3>COPY A:*.*      <- 프로그램을 복사한다.

단, 컴퓨터는 같은 이름의 파일이 존재할 경우 *.EXE 파일 보다
*.COM을 먼저 실행하므로 구 버전인 V3.COM과 V3RES.COM이 존재
하면 신 버전인 V3.EXE와 V3RES.EXE가 실행되지 않습니다. 

따라서 파일 찾기 유틸리티 등을 이용하여 구 버전의  V3.COM과
V3RES.COM을 찾아 반드시 지워야만 합니다. 만약 구버전의 파일
들이 'C:\V3' 디렉토리에 있다고 가정하면  다음과 같은 방법으
로 지울 수 있습니다.

  C:\V3>DEL V3.COM      <- 구 버전의 V3.COM을 지운다.
  C:\V3>DEL V3RES.COM   <- 구 버전의 V3RES.COM를 지운다.

V3+는 현재 시스템 환경을 판단하여 메시지를 출력할 때 사용할
언어를 자동으로 선택합니다. 즉, 영문 도스에서는 영문 메시지
를 출력하고, 한글 도스에서는 한글 메시지를 출력합니다. 그러
나 컴퓨터에 따라서 읽을 수 없는 메시지를 출력하는 경우가 있
을 수 있습니다.

이경우 V3+를 실행시킬 때 /L 옵션을 사용하여 사용언어를 지정
해 주시기 바랍니다.

* 경로(PATH) 지정
현재의 디렉토리 위치와  관계없이 V3+를 쉽게 실행시키고 싶을
때는, AUTOEXEC.BAT의 PATH 명령을 사용하여 V3+가 존재하고 있
는 드라이브 이름과 디렉토리를 추가하시기 바랍니다.

예를들어, V3+가 C:의 \V3 디렉토리에 존재한다면 AUTOEXEC.BAT
내에서 다음과 같이 PATH 명령을 추가하면 됩니다.
 
  PATH=C:\V3


                 V. V3.EXE 사용 방법

1. 기본 사용 방법
V3.EXE는 지금까지 국내에서 발견된  국내외 컴퓨터 바이러스들
을 진단하고 치료해주는 프로그램입니다. 기본적인 사용법은 도
스 프롬프트 상태에서 다음과 같이 입력하면 됩니다.
 
  C:\>V3   

2. 옵션(OPTION) 지정
V3.EXE를 실행시킬 때 사용할 수 있는 옵션은 다음과 같습니다.
 
  V3 [드라이브들][경로들][파일들] [/L:E|K1|K2] [/F|/S|/A|/?]

아무 옵션도 사용하지 않거나  /? 옵션을 사용한 경우에는 기본
으로 기억 장소를 검사  치료하고,  스스로를 검사 치료한 다음
사용 형식을 보여 줍니다. 

드라이브만 지정한 경우에는  먼저 기억 장소와 자기 자신을 검
사하고 플로피 디스크의 경우는 부트 섹터, 하드 디스크의 경우
는 주 부트 섹터와 도스 부트 섹터를 검사한 뒤  모든 디렉토리
의 실행 파일들을 검사합니다.

경로 이름(pathname)을 지정한 경우는 그 디렉토리의 실행 파일
들을 검사합니다. 경로 이름은 주 디렉토리를 포함한 완전한 경
로 이름을 사용하거나 현 디렉토리를 기준으로 한  상대적인 경
로 이름을 모두 사용할 수 있습니다.

파일 이름(filename)을 지정한 경우는 그 파일만을 검사합니다. 
파일 이름,  확장자(extension)에는 와일드 카드(wild card, ?
또는 *)를 사용할 수 있습니다. 


/L 
/L은 V3.EXE에서 메시지를 출력할 때  사용할 언어(language)를
지정해 주는 옵션입니다.  V3.EXE는 현재 상태를 판단해서 메시
지에 사용할 언어를 자동적으로 선택하는데(영문 도스에서는 영
문 메시지, 한글 도스에서는 완성형 한글 메시지를 출력), 컴퓨
터에 따라서 읽을 수 없는 메시지를 출력하는 경우가 간혹 있습
니다.

이러한 경우에는 /L 옵션을 사용하여 사용 언어를 지정해주어야
합니다. /L 옵션의 사용 방법은 다음과 같습니다.
 
  V3 /L:E                    <- 영어로 출력
  V3 /L:K1                   <- 조합형 한글로 출력
  V3 /L:K2                   <- 완성형 한글로 출력

/F
/F는 여러 장의 플로피 디스크를 간편하게 검사할 수 있는 유용
한 옵션입니다. 이 옵션은 플로피 디스크 검사 전에 '검사할 디
스크를 넣고 <Enter> 키를 누르십시오' 라는 메시지를 출력하고
<Enter> 키의 입력을 기다립니다. 

플로피 디스크를 검사할 때마다 매번 명령어를 입력해야만 하는
불편함을 덜 수 있는 옵션입니다.

/S
/S는 경로 이름을 지정해서 검사하는 경우에 해당 디렉토리뿐만
아니라 서브 디렉토리의 파일들도 검사합니다. 드라이브만을 지
정해서 검사하는 경우에는  자동적으로 모든 서브 디렉토리들의
파일들을 검사해 주므로 이 옵션을 사용하지 않아도 됩니다. 서
브 디렉토리가 많은 경우에 유용한 옵션입니다.

/A
/A는 모든 파일을 검사하라는 옵션입니다.  /A 옵션을 사용하지
않고 검사했을 경우  파일 확장자가 COM, EXE, OV?, SYS, BIN인
파일들만 검사하지만,  /A 옵션을 사용하면 다른 확장자를 가진
모든 파일들을 검사합니다. 

처음 실행하는 프로그램을 검사할 때는  /A 옵션을 반드시 사용
하시기 바랍니다. /A 옵션을 사용하지 않은 상태에서 컴퓨터 바
이러스가 발견되었다면 /A 옵션을 사용해서 다시 한 번 해당 디
스크를 검사하시기 바랍니다.


                 VI. V3RES.EXE 사용 방법

1. 기본 사용 방법
V3RES.EXE는 기억 장소에 상주하면서 지금까지 국내에서 발견되
었던 국내외의 컴퓨터 바이러스들을 예방해 주는  프로그램입니
다. 기본적인 사용법은 도스 프롬프트 상태에서 다음과 같이 입
력하면 됩니다.
 
  C:\>V3RES  

2. 옵션(OPTION) 지정
V3RES를 실행시킬 때 사용할 수 있는 옵션은 다음과 같습니다.
 
  V3RES [/L:E|K1|K2] [/U|/?]

/L
/L은 V3RES.EXE에서 메시지를 출력할 때, 사용할 언어(langua-
ge)를 지정해 주는 옵션입니다.  V3RES.EXE는 현재 시스템 환경
을 판단해서 메시지에 사용할 언어를 자동으로 선택하는데(영문
도스에서는 영문 메시지,  한글 도스에서는 완성형 한글 메시지
를 출력합니다.), 컴퓨터에 따라서 읽을 수 없는 메시지를 출력
하는 경우가 간혹 있습니다.  

이러한 경우는  /L 옵션을 사용하여 사용 언어를 지정해 주어야
합니다.  /L 옵션의 사용 방법은 다음과 같습니다.
 
  V3RES /L:E                 <-  영어로 출력
  V3RES /L:K1                <-  조합형 한글로 출력
  V3RES /L:K2                <-  완성형 한글로 출력  

/L을 제외한 다른 옵션을 지정하지 않은 경우 V3RES.EXE 프로그
램이 기억 장소에 상주하여 메시지를 자동으로 선택해 출력합니
다. V3.CFG에 V3RES.EXE의 실행 환경을 정할 수 있는 유일한 옵
션입니다.

/U
/U는 기억 장소에 상주하고 있는 V3RES를 기억 장소에서 제거할
때 사용하는 옵션입니다.

/?
/?는 사용 가능한 옵션들을 알려 주는 사용법을 출력합니다.

2. CONFIG.SYS, AUTOEXEC.BAT
V3RES.EXE가 실행되는 위치는  CONFIG.SYS, AUTOEXEC.BAT, 도스
프롬프트 상태의 세 가지가 있지만, 되도록 도스 상태에서 실행
시키기 보다는 CONFIG.SYS나 AUTOEXEC.BAT에서 자동적으로 실행
되도록 만드는 것이 바람직합니다. 

CONFIG.SYS에서 프로그램을 실행시키기 위해서는 도스 4.0 이상
을 사용해야 합니다.  CONFIG.SYS에서 실행시킬 때는 AUTOEXEC.
BAT에서 실행시킬 때에 비해서, 컴퓨터 바이러스가 먼저 실행될
가능성이 줄어들기 때문에 컴퓨터 바이러스에 대한 예방 효과가
더욱 높아진다는 장점이 있습니다. 

다만 도스 4.x, 도스 5.x에서는 상위기억 장소(upper memory)를
이용할 수 없기 때문에 사용이 가능한 일반 기억 장소(convent-
ional memory)의 크기가 줄어드는 단점이 있습니다. 

AUTOEXEC.BAT에서 프로그램을 실행시킬 때는 도스 5.0 이상에서
는 상위 기억 장소를 이용할 수 있다는 장점이 있습니다.  반면
컴퓨터 바이러스가 먼저 실행될 가능성이 더 높아지므로 이러한
점을 잘 비교해 보고 사용자가 자신의 상황에 맞는 방법을 선택
하는 것이 좋습니다.  

CONFIG.SYS에서는 INSTALL, INSTALLHIGH 명령으로  V3RES.EXE를
기억 장소에 상주시킬 수 있습니다.   INSTALL 명령은 도스 4.0
이상에서 사용할 수 있으며, 일반 기억 장소에 상주시키는 명령
입니다. 

INSTALLHIGH 명령은 도스 6.0 이상에서만 사용 가능하며,  상위
기억 장소에 상주시키는 명령입니다.  이 명령들을 사용할 때는
프로그램의 확장자(extension)까지 지정해 주어야 합니다. 예를
들어, V3RES.EXE 프로그램이 C 드라이브의 \V3 디렉토리에 존재
한다면 CONFIG.SYS에 다음의 INSTALL 명령을 사용하시기 바랍니
다.
 
  INSTALL=C:\V3\V3RES.EXE /L:E   <- 영어로 출력

만일 메시지가 읽을 수 없는 형태로 출력된다면 여기에 /L 옵션
을 추가하여 사용하기 바랍니다. AUTOEXEC.BAT에서 V3RES.EXE를
실행시킬 때는, 도스 쉘 프로그램(DOS shell program)보다는 먼
저 수행되어야 하며, V3RES 이전에 수행되는 프로그램들이 컴퓨
터 바이러스에 감염되지 않았는지 확인해야 합니다. 

AUTOEXEC.BAT에서 실행시키는 방법에는  일반 기억 장소에 상주
시키는 방법과 상위 기억 장소에 상주시키는 방법이 있습니다. 

일반 기억 장소에 상주시킬 때는 프로그램이 존재하는 디렉토리
와 프로그램의 이름만 사용하며, 메시지가 읽을 수 없는 형태로
출력되는 경우에만 /L 옵션을 추가하여 사용하면 됩니다. 

예를들어, V3RES 프로그램이 C 드라이브의 \V3 디렉토리에 존재
한다면 AUTOEXEC.BAT에 다음과 같은 명령만 포함하면 됩니다.
 
  C:\V3\V3RES 

V3RES를 상위 기억 장소에 상주시킬 때는 도스 5.0 이상의 경우
는 LH 또는 LOADHIGH 명령을 사용하며, QEMM의 경우는 LOADHIGH
명령을 사용하면 됩니다.  또한, 메시지가 읽을 수 없는 형태로
출력되는 경우에만 프로그램 이름뒤에 /L 옵션을 추가하여 사용
하면 됩니다. 

예를 들어, 도스 5.0 이상을 사용할 때  V3RES 프로그램이 C 드
라이브의 \V3 디렉토리에 존재한다면 AUTOEXEC.BAT에 다음과 같
은 명령을 사용하면 됩니다.
 
  LH C:\V3\V3RES

충분한 상위 기억 장소가 남아 있음에도 불구하고 V3RES.EXE 가
상위 기억 장소에 상주하지 않는 경우는,  AUTOEXEC.BAT 안에서
실행순서를 바꾸어 주면 상위 기억장소에 상주하는 경우도 있습
니다. 


               VII. V3.EXE(진단.치료용) 메시지 설명

1. 기본적인 메시지
V3.EXE를 실행시키면  먼저 프로그램의 이름과 버전이 출력됩니
다.  만일 등록하지 않은 버전일 경우라면 "This program is an
unregistered version of shareware." (이 프로그램은 셰어웨어
의 비등록판입니다.)라는 말이 출력됩니다. 

V3.EXE는 가장 먼저 시스템의 날짜를 검사합니다.  만약 V3.EXE
가 발표된지 2달 이상 경과되었다면,  "This program is out of
date and should be replaced by new version." (이 프로그램은
유효 기간이 지났습니다. 새로운 버전으로 교환하시기 바랍니다.
)라는 말이 출력됩니다. 

이러한 말이 나올 경우에는 새로운 버전의  V3.EXE를 구해 사용
해야만 신종 컴퓨터 바이러스의 피해를 최소화할 수 있습니다. 

다음은 기억 장소를 검사합니다.  만일 기억 장소에서 지금까지
발견된 컴퓨터 바이러스가 존재하지 않으면 다음과 같은 메시지
를 출력합니다.
 
  Checking Memory : No (known) computer viruses
  기억 장소 검사 : (알려진) 컴퓨터 바이러스는 존재하지 않음

V3.EXE는 기억 장소에서 컴퓨터 바이러스를 발견할 경우,  다음
과 같은 메시지를 출력하면서, 자동으로 기억 장소에 있는 컴퓨
터 바이러스를 제거합니다.
 
  Checking Memory : Brain VIRUS found -> Repaired
  기억 장소 검사 : Brain VIRUS 발견   -> 고쳤음

기억 장소를 검사한 다음에는 V3.EXE 프로그램 자체를 검사합니
다.  만일 프로그램 자체에 이상이 없다면, 다음과 같은 메시지
를 출력합니다.
 
  Checking V3.EXE : No (known) computer viruses
  V3.EXE 검사 : (알려진) 컴퓨터 바이러스는 존재하지 않음

그런데 V3.EXE가  '센트럴 포인트 안티바이러스(Central Point
Anti-Virus)'나 '터보 안티바이러스(Turbo Anti-Virus)'의 면역
(immunization) 기능에 의해 면역된 경우는 경고음과 함께 다음
과 같은 메시지가 출력되면서 실행을 중단합니다.
 
  Checking V3.EXE :
  This program is immunized by Central Point Anti-Virus or
  Turbo Anti-Virus. Please remove the immunization.
  V3.EXE 검사 :
  이 프로그램에 설치된 Central Point Anti-Virus 또는 
  Turbo Anti-Virus의 면역 기능을 제거해 주십시오.

이럴 경우에는 다시 '센트럴 포인트 안티바이러스' 나 '터보 안
티 바이러스'를 사용하여  면역 기능을 제거해 주어야만 실행이
가능합니다.

만약 V3.EXE 자체가 진단 및 치료 가능한 컴퓨터 바이러스에 감
염되었을 경우 자신이 스스로 감염된 컴퓨터 바이러스를 진단하
고 치료합니다.
 
  Checking V3.EXE : Jerusalem VIRUS found -> Repaired
  V3.EXE 검사 : Jerusalem VIRUS 발견      -> 고쳤음

그러나 V3.EXE 자체가 새로운 컴퓨터 바이러스에 감염된 경우에
는 경고음과 함께 다음과 같은 메시지를 출력하며,  이 때 아무
키나 누르면 프로그램의 실행을 중단합니다.
 
  Checking V3.EXE :
  This program is probably infected with unknown computer virus.
  Please send this program to Ahn's Anti-Virus Laboratory.

  V3.EXE 검사 :
  이 프로그램은 새로운 컴퓨터 바이러스에 감염되어 있습니다.
  이 프로그램을 안철수컴퓨터바이러스연구소로 보내 주십시오.

이 경우에는 V3.EXE가 새로운 컴퓨터 바이러스에 감염된 경우이
므로 즉시 V3+의 사용을 중지하시고 감염된 V3.EXE 프로그램 및
기타 감염된 프로그램을  컴퓨터 통신이나 우편을 통해 '안철수
컴퓨터바이러스연구소'로 보내 주시기 바랍니다.

감염된 상태에서 계속해서 컴퓨터를 사용한다면 더욱 큰 피해를
입을 수 있습니다. 보내주신 샘플은 신종 컴퓨터 바이러스 여부
를 확인한 후, 신속한 백신 프로그램 버전업으로 컴퓨터 사용자
의 피해를 최소화할 것입니다. 안철수연구소의 연락처는 설명서
첫부분에 명기되어 있습니다.


2. 사용에 관한 메시지
V3+를 아무 옵션도 사용하지 않았거나 '/?' 옵션을 사용해 실행
한 경우 다음과 같은 도움말이 출력됩니다.
 
  Usage: V3 [drives][\paths][files] [options]
       (Options) /L  (L)anguage
                     /L:E   (E)nglish
                     /L:K1  (K)orean - combination code
                     /L:K2  (K)orean - KS complete code
                 /F  check several (F)loppy disks
                 /S  check lower (S)ubdirectories
                 /A  check (A)ll files
                 /?  show this help message

  사용법: V3 [드라이브들][\경로들][파일들] [옵션들]
        (옵션)  /L  사용 언어
                    /L:E   영어
                    /L:K1  조합형 한글
                    /L:K2  완성형 한글
                /F  여러 장의 플로피 디스크 검사
                /S  서브 디렉토리 검사
                /A  모든 파일 검사
                /?  사용법 출력

검사할 드라이브, 경로 또는, 파일을 지정했을 경우에는 다음과
같이 검사할 대상을 출력하고 검사를 시작합니다.
 
  CHECKING C:
  C: 검사

플로피 디스크의 경우에는 부트 섹터와 파일들을 검사하며,  하
드 디스크의 경우에는 주 부트 섹터(master boot sector), 도스
부트 섹터(DOS boot sector), 파일들을 검사합니다. 그 밖의 디
스크에 대해서는 파일들만 검사합니다.

/F 옵션을 사용하여 플로피 디스크를 검사할 경우, 디스크를 검
사하기 전 다음과 같은 메시지를 출력하고  디스크를 넣기를 기
다립니다. 검사할 플로피 디스크마다 매번 명령어를 입력해야만
하는 불편을 덜 수 있는 편리한 옵션입니다.
 
  Insert a disk in drive and press <Enter>
  검사할 디스크를 넣고 <Enter> 키를 누르십시오.

하드 디스크의 주 부트 섹터를 검사할 때는 'Master Boot Sect-
or :' (주 부트 섹터 :)가 출력되고 하드 디스크의 도스 부트섹
터나 플로피 디스크의 부트섹터를 검사할 때는 'MS-DOS Boot S-
ector :'(도스 부트 섹터 :)가 출력됩니다. 

주 부트 섹터에서 지금까지 발견되었던 컴퓨터 바이러스들이 존
재하지 않는 경우에는 다음과 같은 말을 출력합니다.
 
  Master Boot Sector : No (known) computer viruses
  주 부트 섹터 : (알려진) 컴퓨터 바이러스는 존재하지 않음

만일 주 부트 섹터에서 컴퓨터 바이러스를 발견한 경우, 경고음
과 함께 컴퓨터 바이러스의 이름을 출력한 다음, 치료할 것인지
를 묻습니다.  이 때, <Y> 키를 누르면 컴퓨터 바이러스를 치료
하고 'Repaired'(고쳤음)라는 말을 출력하며, <N> 키나 <Enter>
키를 누르면 치료하지 않고 다음으로 넘어갑니다.
 
  Master Boot Sector : Brain VIRUS found - Repair it? (y/N)
  주 부트 섹터 : Brain VIRUS 발견 - 고치겠습니까? (y/N)

파일 검사 중  컴퓨터 바이러스를 발견하면 경고음과 함께 컴퓨
터 바이러스의 이름을 출력한 다음에 치료할 것인지를 묻습니다. 

이때 <Y> 키를 누르면 컴퓨터 바이러스를 치료한 뒤 'Repaired'
(고쳤음)라는 말을 출력하며, <N> 키나 <Enter> 키를 누르면 치
료하지 않고 다음으로 넘어갑니다.
 
  Checking C:\VIRUS\SAMPLE\DAV.COM
              Dark Avenger VIRUS found - Repair it? (y/N)

  Checking C:\VIRUS\SAMPLE\DAV.COM
              Dark Avenger VIRUS 발견 - 고치겠습니까? (y/N)

만약, 겹쳐쓰기 바이러스(overwriting virus) 등과 같이 파일의
복구가 불가능한 컴퓨터 바이러스를 발견하면 경고음과 함께 발
견된 컴퓨터 바이러스의 이름과 "- Repair it? (y/N)"(고치겠습
니까? (y/N)) 라는 메시지를 출력하며,  <N> 키나 <Enter> 키를
누르면 지우지 않고 다음으로 넘어갑니다. 

<Y>키를 눌렀을 경우  "This file cannot be repaired - DELETE
it (y/N)"(이 파일은 고칠 수 없습니다. - 지우겠습니까? (y/N))
라는 메시지를 출력하며 <Y>키를 누르면 감염된 파일을 지운 후
'DELETED'(지움)라는 말을 출력하며  <N> 키나 <Enter> 키를 누
르면  위와 마찬가지로 감염 파일을 지우지 않고 다음으로 넘어
갑니다.
 
  Checking C:\VIRUS\SAMPLE\LEPROSY.COM
              Leprosy VIRUS found - Repair it? (y/N) Y
              This file cannot be repaired. - DELETE it? (y/N)

  Checking C:\VIRUS\SAMPLE\LEPROSY.COM
              Leprosy VIRUS 발견 - 고치겠습니까? (y/N) Y
              이 파일은 고칠 수 없습니다. - 지우겠습니까? (y/N)

아울러 V3+에는 국내에서 아직 발견되지 않은 외국 컴퓨터 바이
러스에 대한 예방 진단 기능이 포함되어 있습니다. 만일 이러한
컴퓨터 바이러스가 발견될 경우 다음의 메시지가 출력됩니다. 

이런 경우 국내에서 처음 발견된 컴퓨터 바이러스이므로 안철수
컴퓨터바이러스연구소에 연락해 주시면 해당 컴퓨터 바이러스에
대한 치료 기능을 포함토록 하겠습니다.
 
  This is a NEW computer virus.
  Please inform Ahn's Anti-Virus Laboratory.

  이것은 국내 미발견 컴퓨터 바이러스입니다.
  안철수컴퓨터바이러스연구소로 연락하십시오.

파일들을 검사하여 감염된 파일이 없을때는 다음과 같은 메시지
를 출력합니다.
 
  Directories & Files: No (known) computer viruses

감염된 파일들이 있을 때는 감염된 파일의 수를 출력합니다.
 
  Directories & Files: 6 infected files

만약 /A 옵션을 사용하지 않은  상태에서 감염된 파일들을 발견
했다면, 마지막에 다음과 같은 메시지를 출력합니다.

  in all files.

  실행 파일 이외에도 컴퓨터 바이러스가 존재할 수 있습니다.
  /A 옵션을 사용해서 다시 검사하시기 바랍니다.


                 VIII. V3RES.EXE(예방용) 메시지 설명

1. 기본적인 메시지

V3RES.EXE를 실행시키면 먼저 프로그램의 이름과 버전이 출력됩
니다.  그 다음에 등록하지 않은 버전일 경우 "This program is
an unregistered version of shareware."(이 프로그램은 셰어웨
어의 비등록판입니다.)라는 말이 출력됩니다.

V3RES.EXE는 시스템 날짜를 검사합니다. 만약 V3RES.EXE가 제작
된 지 2달 이상이 경과되었다면, "This program is out of date
and should be replaced by new version." (이 프로그램은 유효
기간이 지났습니다. 새로운 버전으로 교환하시기 바랍니다.) 라
는 말이 출력됩니다.

이러한 말이 나올 경우에는 새로운 버전의  V3RES.EXE를 구해서
사용해야만 신종 컴퓨터 바이러스의 피해를 최소화할 수 있습니
다.

다음은 기억 장소를 검사합니다.  만일 기억 장소에서 지금까지
발견된 컴퓨터 바이러스가 존재하지 않으면,  다음과 같은 메시
지를 출력합니다.
 
  Checking Memory : No (known) computer viruses
  기억 장소 검사 : (알려진) 컴퓨터 바이러스는 존재하지 않음

만약 V3RES.EXE가 기억 장소에서 컴퓨터  바이러스를 발견할 경
우 경고음과 함께 다음과 같은 메시지를 출력하며,  이 때 아무
키나 누르면 프로그램의 실행이 중단됩니다.
 
  Checking Memory : Brain VIRUS found
  Please run V3 to cure it.

  Press any key to stop the program.

  기억 장소 검사 : Brain VIRUS 발견
  V3를 사용하여 치료하십시오.

  아무 키나 누르면 프로그램이 중단됩니다.

이 경우에는 V3.EXE를 실행시켜 컴퓨터 바이러스를 치료해야 합
니다. 기억 장소를 검사한 다음 V3RES.EXE  프로그램 자체를 검
사합니다. 만일 프로그램 자체에 이상이 없다면 다음과 같은 메
시지를 출력합니다.
 
  Checking V3RES.EXE : No (known) computer viruses
  V3RES.EXE 검사 : (알려진) 컴퓨터 바이러스는 존재하지 않음

그런데 만일 V3RES.EXE 프로그램이  '센트럴 포인트 안티바이러
스(Central Point Anti-Virus)'나 '터보 안티바이러스(Turbo A-
nti-Virus)'의 면역(immunization)기능에 의해 면역된 경우에는
경고음과 함께 다음과 같은 메시지가 출력되면서 실행을 중단합
니다.
 
  Checking V3RES.EXE :
  This program is immunized by Central Point Anti-Virus or
  Turbo Anti-Virus. Please remove the immunization.

  V3RES.EXE 검사 :
  이 프로그램에 설치된 Central Point Anti-Virus 또는 
      Turbo Anti-Virus의 면역 기능을 제거해 주십시오.

이 경우에는 다시 '센트럴 포인트 안티바이러스' 나 '터보 안티
바이러스' 를 사용해서 면역 기능을 제거해 주어야만 V3RES.EXE
의 실행이 가능합니다.

만일, V3RES.EXE 프로그램 자체가 새로운 컴퓨터 바이러스에 감
염된 경우는 경고음과 함께 다음과 같은 메시지를 출력하며, 이
때 아무 키나 누르면 프로그램의 실행을 중단합니다.
 
  Checking V3RES.EXE :
  This program is probably infected with unknown computer virus.
  Please send this program to Ahn's Anti-Virus Laboratory.

  Press any key to stop the program.

  V3RES.EXE 검사 :
  이 프로그램은 새로운 컴퓨터 바이러스에 감염되어 있습니다.
  이 프로그램을 안철수컴퓨터바이러스연구소로 보내 주십시오.

  아무 키나 누르면 프로그램이 중단됩니다.

이 경우에는 V3RES.EXE가 새로운 컴퓨터  바이러스에 감염된 경
우 이므로 즉시 V3+의 사용을 중지하고 감염된 V3RES.EXE 및 기
타 감염된 프로그램을 컴퓨터 통신이나 우편을 통해서  '안철수
컴퓨터바이러스연구소'로 보내 주시기 바랍니다. 감염된 상태에
서 계속해 컴퓨터를 사용한다면 더욱 큰 피해를 입을 수 있습니
다.

사용자 여러분께서 보내 주신 샘플은 신종 컴퓨터 바이러스 여
부를 확인한 후 신속한 백신 프로그램 버전업으로  컴퓨터 사용
자의 피해를 최소화할 것입니다.  연락처는 이 설명서의 첫부분
에 명기되어 있습니다.

2. 사용에 관한 메시지
V3RES.EXE는 기억장소와 프로그램을 검사한 다음에는  사용자가
지정한 옵션에 따라 진행됩니다. 아무 옵션도 사용하지 않은 경
우는 다음과 같은 말을 출력하면서 기억 장소에 상주합니다.
 
  V3RES is installed in memory.
  V3RES가 기억 장소에 상주했습니다.

V3RES.EXE는 /U 옵션을 사용해 실행한 경우,  다음과 같은 말을
출력하면서 상주하고 있던 V3RES.EXE가 기억 장소에서 제거됩니
다.
 
  V3RES is uninstalled in memory.
  V3RES가 기억 장소에서 제거되었습니다.

V3RES.EXE를 /? 옵션을 사용해 실행한 경우, 사용할 수 있는 옵
션들을 출력하면서 기억 장소에 상주하지 않고 실행을 종료합니
다.
 
  Usage: V3RES [/L:E|K1|K2] [/U|/?]
                /L  (L)anguage
                    /L:E   (E)nglish
                    /L:K1  (K)orean - combination code
                    /L:K2  (K)orean - KS complete code
                /U  (U)ninstall V3RES in memory
                /?  show this help message

  사용법: V3RES [/L:E|K1|K2] [/U|/?]
                 /L  사용 언어
                     /L:E   영어
                     /L:K1  조합형 한글
                     /L:K2  완성형 한글
                 /U  기억 장소에서 제거
                 /?  사용법 출력

일단 V3RES.EXE가 기억 장소에 상주한 다음에는, 지금까지 발견
되었던  컴퓨터 바이러스에 감염된 프로그램을 실행시키려 해도
다음과 같이 감염된 컴퓨터 바이러스 이름이 출력되면서 프로그
램이 실행되지 않습니다.
 
  This file is infected with Jerusalem VIRUS.
  Please run V3 to cure it.

  이 파일은 Jerusalem VIRUS에 감염되었습니다.
  V3.EXE로 치료하십시오.

이런 경우에는 V3.EXE를 실행시켜서  컴퓨터 바이러스에 감염된
해당 파일을 치료해야 하고, 다시 /A 옵션을 이용하여 드라이브
전체를 검사하시기 바랍니다.


                 IX.   오류 메시지 설명

V3.EXE, V3RES.EXE의 실행 도중에 오류가 발생했을 경우는 다음
의 오류 메시지들을 출력하며 "Continue anyway?" (계속 진행할
까요?) 라고 묻습니다.  이 때 <Y> 키를 누르면 실행을 계속 진
행하며, <N> 키나 <Enter> 키를 누르면 실행을 종료합니다.

  This program requires DOS 3.2 or above. (V3.EXE,V3RES.EXE
에서) V3.EXE, V3RES.EXE를 사용하기 위해서는 도스 3.2 이상의
버전을 사용해야 합니다.

  ERROR: Cannot change directory. (V3.EXE에서)
(오류 발생: 디렉토리로 이동할 수 없습니다.)
이러한 오류 메시지는 도스가 오동작을 하거나 디렉토리 구조가
잘못되었을 때 발생합니다. 다시 부팅시킨 후 검사해도 계속 이
메시지가 나올 때는 NDD(노턴 디스크 닥터)와 같은  유틸리티로
디스크를 검사해 보는 것이 좋습니다.

  ERROR: Cannot delete file. (V3.EXE에서)
(오류 발생: 파일을 지울 수 없습니다.)
네트워크에서 권한이 없는 파일을 지우려고 할 때 발생합니다.

  ERROR: Cannot find the original boot sector. (V3.EXE에서)
(오류 발생: 원래의 부트 섹터를 찾을 수 없습니다.)
원래의 부트 섹터도 파괴되어 있거나, 원래의 부트 섹터를 다른
위치에 보관하는 변형 부트 바이러스일 때 발생합니다.  부팅이
되지 않는 경우에는 NDD(노턴 디스크 닥터) 등의 유틸리티로 치
료해 보고, 부팅이 되는 경우에는 변형 부트 바이러스일 가능성
이 높습니다. '안철수컴퓨터바이러스연구소'로 연락주시기 바랍
니다.(연락처는 이 설명서의 첫 부분에 명기되어 있습니다).

  ERROR: Cannot open file. (V3.EXE에서)
(오류 발생: 파일을 사용할 수 없습니다.)
CONFIG.SYS에서 FILES 를 충분히 지정하지 않았을 때 발생할 수
있으며, 이 때는 FILES=30 이상 잡아 주어야 합니다. 또한 파일
이름이 잘못되었거나, 네트워크에서 권한이 없는 파일을 사용하
려고 할 때도 발생할 수 있습니다.

  ERROR: Cannot read disk. (V3.EXE에서)
(오류 발생: 디스크를 읽을 수 없습니다.)
잘못해서 디스크 드라이브의 문을 닫지 않았거나,  손상된 디스
크를 사용하거나, 포맷되지 않은 디스크를 사용할 때 발생합니다.

  ERROR: Cannot read file. (V3.EXE에서)
(오류 발생: 파일을 읽을 수 없습니다.)
잘못해서 디스크 드라이브의 문을 닫지 않았거나 손상된 디스크
나 파일을 읽을 때 발생합니다.

  ERROR: Cannot set file attribute. (V3.EXE에서)
(오류 발생: 파일 속성을 바꿀 수 없습니다.)
램상주 백신 프로그램을 사용하거나, 네트워크에서 권한이 없는
파일을 치료하기 위해 파일 속성을 바꾸려고 할 때 발생합니다.

  ERROR: Cannot write disk. (V3.EXE에서)
(오류 발생: 디스크에 쓰기를 할 수 없습니다.)
쓰기 방지 탭(write protect tab)을 붙인 상태이거나 손상된 디
스크를 사용할 때 발생합니다.

  ERROR: Cannot write file. (V3.EXE에서)
(오류 발생: 파일에 쓰기를 할 수 없습니다.)
쓰기 방지 탭(write protect tab)을 붙인 상태이거나 손상된 디
스크를 사용할 때 발생합니다.

  ERROR: Drive is invalid. (V3.EXE에서)
(오류 발생: 드라이브가 존재하지 않습니다.)
존재하지 않는 드라이브를 지정했을 때 발생합니다.

  ERROR: Insufficient memory. (V3.EXE에서)
(오류 발생: 기억 장소가 부족합니다.)
램상주 프로그램이 많은 상태에서 V3.EXE를 실행하면 발생합니다.

  ERROR: Option is incorrect. Use /? to view options. 
   (V3.EXE, V3RES.EXE에서)
(오류 발생: 잘못된 옵션을 사용했습니다.  /?을 사용하면 정확
 한 옵션을 볼 수 있습니다.) 
V3.EXE에서 /L, /F, /S, /A, /?  이외의 옵션을 사용했을 때 발
생하며, V3RES.EXE에서는 /L, /U, /? 이외의 옵션을 사용했을때
발생합니다.

  ERROR: Pathname is invalid. (V3.EXE에서)
(오류 발생: 경로 이름이 존재하지 않습니다.)
잘못된 경로 이름이나 파일 이름을 지정했을 때 발생합니다.

  ERROR: V3RES cannot be uninstalled in memory.(V3RES.EXE에서)
(오류 발생: V3RES를 기억 장소에서 제거할 수 없습니다.)
V3RES.EXE가 기억 장소에 상주한 다음, 다른 기억 장소 상주 프
로그램이 또 다시 상주한 경우에는,  /U 옵션을 사용해도 V3RES.
EXE를 기억 장소에서 제거할 수 없습니다. 이 경우에는 다른 기
억 장소 상주 프로그램을 먼저 기억장소에서 제거한 다음 V3RES.
EXE를 제거하면 됩니다.

  ERROR: V3RES is already installed in memory.(V3RES.EXE에서)
(오류 발생: V3RES가 기억 장소에 이미 존재하고 있습니다.)
V3RES.EXE가 기억장소에 이미 상주하고 있는데도 불구하고 다시
V3RES.EXE를 실행시킬 경우 이러한 메시지가 출력됩니다.(V3RES.
EXE는 컴퓨터를 켠 후 한번만 실행하면 됩니다.)

  ERROR: V3RES is not present in memory.(V3RES.EXE에서)
(오류 발생: V3RES가 기억 장소에 존재하지 않습니다.)
V3RES.EXE가 기억 장소에 상주하고 있지 않을 때  /U 옵션을 사
용하면 이러한 메시지가 출력됩니다.


                 X. V3BACKUP.EXE 사용 방법

1. V3BACKUP.EXE란?
V3BACKUP.EXE는 사용자가 편리하게 디스크 정보의 백업 및 부트
바이러스를 채취할 수 있는 유틸리티입니다.  새로운 부트 바이
러스에 감염되었을 경우 감염되지 않은 깨끗한 플로피 디스켓으
로 부팅한 다음 본 유틸리티를 이용하여 감염된 부트 섹터를 파
일로 만들어서 안철수연구소로 보내주시기 바랍니다.

감염되지 않은 경우라도  V3BACKUP.EXE을 이용하여 부트 섹터를
백업해 놓으면 부트 섹터 복구시 유용한 자료가 됩니다.

2. V3BACKUP.EXE의 사용법
도스 프롬프트 상태에서 다음과 같이 입력하면 지정한 드라이브
(A:, B:, C:)의 주 부트 섹터 및 도스 부트 섹터를 파일로 만들
어 현재의 디렉토리에 저장합니다.

  C:\>V3BACKUP C:

V3BACKUP.EXE를 실행시킬 때 사용할 수 있는 옵션은 /? 옵션 한
가지입니다.  드라이브를 지정하지 않았거나  /? 옵션을 사용한
경우는 사용 형식을 보여줍니다. 부트 섹터 채취를 위해서는 드
라이브 이름을 반드시 지정하여야 하며 지정한 드라이브의 부트
섹터를 채취해 다음과 같은 파일 이름으로 저장합니다. 

또한, 램 드라이브, CD-ROM 드라이브, 네트워크 드라이브 등 부
팅 불가능한 드라이브를 지정했을 경우 에러 메시지를 출력하고
파일을 생성하지 않습니다.

플로피 디스크   Boot Sector             FBS.V3
하드 디스크     Master Boot Sector      MBS.V3
하드 디스크     MS-DOS Boot Sector      DBS.V3


3. V3BACKUP.EXE의 메시지

  Usage: V3BACKUP [drive] [/?]
사용법을 출력하는 메시지입니다. 

드라이브를 지정하지 않았거나 /? 옵션을 사용한 경우 출력됩니
다.

  Boot Sector is saved as FBS.V3
부트 섹터가 FBS.V3로 저장되었습니다.

플로피 디스크의 부트 섹터를 채취한 후 출력하는 메시지입니다.

  Master Boot Sector is saved as MBS.V3
부트 섹터가 MBS.V3로 저장되었습니다.

하드 디스크의 주 부트 섹터를 채취한 후 출력하는 메시지입니
다. 

  MS-DOS Boot Sector is saved as DBS.V3
부트 섹터가 DBS.V3로 저장되었습니다.

하드 디스크의 도스 부트 섹터를 채취한 후 출력하는 메시지입
니다. 

  ERROR: This program requires DOS 3.2 or above.
V3BACKUP.EXE를 사용하기 위해서는 도스 3.2 이상의 버전을 사
용해야 합니다.

  ERROR: There is no boot sector in the drive.
이 드라이브에 부트 섹터가 없습니다.

지정한 드라이브가 부팅 불가능한 드라이브일 경우 발생합니다(
부팅 불가능한 드라이브는 램 드라이브, CD-ROM 드라이브, 네트
워크 드라이브, C:를 제외한 하드 디스크 드라이브 등이 있습니
다.).

  ERROR: Cannot read disk.
디스크를 읽을수 없습니다.

잘못해서 디스크 드라이브의 문을 닫지 않았거나, 손상된 디스
크를 사용하거나, 포맷되지 않은 디스크를 사용할 때 발생합니다.

  ERROR: Cannot create file.
파일을 생성할 수 없습니다. 

이 오류는 쓰기 방지 탭(write protect tab)을 붙인 상태, 손상
된 디스크를 사용할 때,  디스크 용량 부족의 경우,  쓰기 방지 
기능을 갖는 일부 컴퓨터 바이러스에 감염되었을 경우에 발생합
니다.

  ERROR: Cannot write file.
파일에 쓰기를 할 수 없습니다. 

쓰기 방지 탭(write protect tab)을 붙인 상태이거나 손상된 디
스크를 사용할 때 발생합니다.
